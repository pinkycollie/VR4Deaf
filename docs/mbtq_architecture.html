Module Naming Convention

All modules follow the @vr4deaf/magician-{category}-{name} pattern:

  • @vr4deaf/magician-engine-accessibility - Accessibility engine
  • @vr4deaf/magician-ui-asl-navigation - ASL navigation UI kit
  • @vr4deaf/magician-agent-career - Career matching agent
  • @vr4deaf/magician-branded-mbtq-enterprise - Enterprise branded module

Visual Identity Guidelines

High Contrast

All visual elements must meet WCAG AAA standards (7:1 contrast ratio minimum).

Visual First

Icons, colors, and visual indicators are primary communication methods.

ASL Friendly

Clear visual hierarchy, no audio dependencies, gesture-friendly interfaces.

Consistent Design

Unified design language across all modules and platforms.

âš™ï¸ Technology Stack

Frontend Stack

  • Framework: Next.js 15+ with App Router
  • UI Library: React 19+
  • Language: TypeScript 5+
  • Styling: TailwindCSS with custom accessibility themes
  • Components: Radix UI primitives, custom Deaf-first components
  • State Management: React Context, Zustand for complex state

Backend Stack

  • Primary API: FastAPI (Python 3.8+)
  • Secondary API: Node.js/Express (TypeScript)
  • Databases: PostgreSQL, MongoDB
  • AI/ML: TensorFlow, PyTorch, custom models
  • Real-time: WebSocket, Server-Sent Events

Developer Tools

  • Package Manager: pnpm (workspace support)
  • Build Tool: TypeScript compiler, Next.js bundler
  • Testing: Vitest, React Testing Library
  • Linting: ESLint with custom accessibility rules
  • CI/CD: GitHub Actions

Module Publishing

  • Public Registry: npm (npmjs.com)
  • Private Registry: npm organization or Verdaccio
  • Versioning: Semantic Versioning (semver)
  • Documentation: README, API docs, TypeScript types

ğŸ”„ Development Workflow

Module Development Process

  1. Planning
    • Define module purpose and scope
    • Create GitHub issue for discussion
    • Design API and interfaces
  2. Development
    • Create module structure in appropriate category
    • Implement core functionality with TypeScript
    • Write comprehensive tests (80%+ coverage)
    • Document API and usage examples
  3. Quality Assurance
    • Run automated tests and linting
    • Verify accessibility standards (WCAG AAA)
    • Test cross-platform compatibility
    • Security audit for vulnerabilities
  4. Review & Approval
    • Submit pull request with detailed description
    • Address code review feedback
    • Obtain maintainer approval
  5. Publishing
    • Version bump following semver
    • Update CHANGELOG.md
    • Publish to npm registry
    • Create GitHub release with notes
  6. Maintenance
    • Monitor issues and feedback
    • Release bug fixes and updates
    • Keep dependencies updated
    • Support community contributions
